# AI 默认提示词优化设计

## 目标

优化 AI 招呼语和 AI 过滤的默认提示词，让新用户不需要先手写 Prompt，也能得到更稳定、克制且可解析的输出。

## 当前缺口

- 默认配置中 `aiGreeting.prompt` 和 `aiFiltering.prompt` 为空，新用户必须手动填写或点击示例。
- 现有示例对输出约束较弱，招呼语容易过长、泛泛而谈或带书信格式。
- AI 过滤示例只给加减分项，没有明确总分基准、硬性风险项和 JSON 纯输出要求，解析稳定性不足。

## 已锁定决策

- 默认启用的提示词仍不自动开启 AI 功能，只作为用户开启 AI 后的初始内容。
- 默认使用单轮字符串提示词，兼容当前 UI 的默认单文本编辑模式。
- AI 招呼语要求输出 1 到 2 句中文聊天开场白，避免承诺、夸大、薪资谈判和占位符。
- AI 过滤要求只输出 JSON，结构保持现有 `positive` 和 `negative` 数组，继续兼容 `parseFiltering` 和分数阈值逻辑。
- AI 过滤把外包、驻场、外派、派遣、OD、客户现场、第三方交付、人力外包、项目制交付作为硬性风险；有明确证据时严重扣分，避免继续投递外包类岗位。
- AI 过滤把实习、校招、应届培养、管培生、毕业生项目、可转正实习作为硬性风险；有明确证据时严重扣分，避免继续投递实习或应届培养类岗位。
- 示例提示词与默认提示词保持一致，避免“默认”和“填入示例”行为不一致。

## 非目标

- 不修改会员模式后端 Prompt。
- 不调整 AI 过滤分数阈值和解析逻辑。
- 不改变已有用户已经保存的配置。
- 不新增 Prompt 模板管理或版本迁移。

## 数据或状态变化

- 仅改变新默认配置和示例内容。
- 已保存过配置的用户继续使用现有本地配置，不强制覆盖。

## API 或页面影响

- 无后端 API 变化。
- AI 配置弹窗中的“填入示例”内容会更新。
- 新安装或清空配置后的默认 AI Prompt 会更新。

## 验收标准

- 新默认 AI 招呼语 Prompt 能约束输出为短、自然、岗位相关的聊天开场白。
- 新默认 AI 过滤 Prompt 能约束模型只返回可解析 JSON。
- 新默认 AI 过滤 Prompt 对明确外包/驻场/外派/派遣/OD/客户现场类岗位给出严重扣分。
- 新默认 AI 过滤 Prompt 对明确实习/校招/应届培养/管培生类岗位给出严重扣分。
- 输出 JSON 仍使用 `positive` 和 `negative` 两个数组，每项包含 `reason` 和 `score`。
- `pnpm check` 和 `pnpm lint` 通过。
