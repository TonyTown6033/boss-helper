# 文档框架初始化设计

## 目标

为 Boss Helper 建立最小可用的文档优先框架，使后续功能开发在进入产品代码前具备任务索引、设计文档入口和可检查的链接关系。

## 当前缺口

- `README.md` 中已有面向用户的功能 TODO，但缺少面向开发协作的独立任务索引。
- `docs/` 当前主要保存截图资源，缺少文档入口和领域目录说明。
- `CLAUDE.md` 已描述技术栈和代码结构，但缺少新增功能时的文档优先流程。

## 已锁定决策

- 使用仓库根目录 `TODO.md` 作为开发任务索引。
- 使用 `docs/README.md` 作为文档入口。
- 使用 `docs/project/` 存放项目级协作和框架说明。
- 后续功能文档优先放入已有领域目录；没有合适领域时，在 `docs/` 下创建短且稳定的领域名。
- 小范围功能至少包含 `<feature>-design.md`，并覆盖目标、当前缺口、锁定决策、非目标、数据或状态变化、API 或页面影响、验收标准。

## 非目标

- 不调整产品功能、构建脚本或浏览器扩展运行逻辑。
- 不迁移 `README.md` 中现有的用户向 TODO。
- 不建立完整测试框架或发布流程。

## 数据或状态变化

无运行时数据、浏览器存储或用户配置变化。

## API 或页面影响

无 API、扩展页面、内容脚本或 UI 影响。

## 实施顺序

1. 新增 `TODO.md`，登记“文档框架初始化”任务并链接本设计文档。
2. 新增 `docs/README.md`，说明文档目录和文档优先流程。
3. 新增 `docs/project/documentation-framework-design.md`，记录本次框架决策。
4. 更新 `CLAUDE.md`，把文档优先流程纳入项目指南。
5. 使用 `feature-doc-first` 的 TODO 链接检查脚本验证任务和文档链接。

## 验收标准

- `TODO.md` 存在，并包含“文档框架初始化”任务。
- `TODO.md` 同一任务行链接到 `docs/project/documentation-framework-design.md`。
- `docs/README.md` 存在，并能作为文档入口说明目录用途。
- `CLAUDE.md` 包含新功能开发前的文档优先约定。
- TODO 链接检查脚本通过。
