# Boss 聊天自动回复设计

## 目标

在 Boss 聊天页为当前打开的会话提供 AI 回复草稿能力。用户启用 `AI回复` 并配置模型后，插件监听当前会话里对方新消息，生成一条简短、克制、求职者视角的回复，并填入 BOSS 聊天输入框，等待用户检查后手动处理。

## 当前缺口

- `README.md` 中已有 `自动回复聊天` 路线项，但当前实现只支持自动打招呼。
- 配置系统已有 `aiReply` 字段和模型选择入口，但 UI 中该开关被禁用，且默认 Prompt 为空。
- `/web/geek/chat` 已有只读 DOM 采集面板，但不会调用模型或发送消息。
- 聊天页需要复用 AI 模型能力生成回复，但不应绕过用户直接发送。

## 已锁定决策

- 初版只处理用户当前打开的会话，不自动遍历联系人列表，不自动点击未读会话。
- 功能默认关闭，只在 `conf.formData.aiReply.enable === true` 时运行。
- 自动草稿只针对方向为 `other` 的最新新消息，忽略本人消息、系统消息和空文本。
- `立即回复` 属于手动触发：优先使用方向为 `other` 的最新消息；如果页面采集未能识别 `direction=other`，允许兜底使用当前可见的最新非本人、非系统文本消息，避免用户明确点击时被方向识别误伤。
- 每条消息最多自动生成一次草稿；去重状态保存在当前页面运行时内存和 `sessionStorage` 中，降低刷新页面后的重复生成风险。
- 发送前遵守 `conf.formData.delay.messageSending` 秒延迟，让页面状态稳定，也降低连续发送风险。
- AI Prompt 使用 `aiReply.prompt`，模板上下文包含当前会话标题、最近消息、最近聊天记录、当前岗位卡片、当前岗位详情页摘要、求职者资料摘要、AI筛选标准、当前时间和页面 URL。
- AI筛选标准只作为求职偏好和岗位风险判断参考，用于决定应该追问哪些岗位信息；AI回复不得继承 AI筛选的 JSON 输出格式。
- AI Prompt 运行时会追加回复质量约束：优先正面回答对方问题，给出资料中可验证的具体点；回复目标是获取更多岗位有效信息，不做无条件顺从式确认，只有资料缺失时才说明需要确认，且不能用空泛套话替代回答。
- 对方要求附件简历、微信、电话或资料时，草稿可以简短回应，但必须顺带询问 1 个岗位关键问题，例如核心职责、技术栈侧重点、团队/项目方向、工作模式或面试流程。
- 自动回复上下文窗口取当前已采集聊天记录最近 20 条；不会主动滚动加载更早历史。
- 自动回复只生成纯文本草稿。内容为空时不填入输入框。
- 插件不直接调用发送接口、不点击发送按钮、不模拟 Enter 发送；最终发送或修改由用户决定。
- VIP 签名模型当前没有 `aiReply` 后端接口，初版仅支持用户自行配置的 OpenAI 兼容模型。

## 非目标

- 不绕过登录、验证码、风控、反爬或平台限制。
- 不模拟人类持续多轮策略，不做批量私信。
- 不主动滚动加载聊天历史，不解析 WebSocket 历史包。
- 不自动切换会话、不自动滚动加载历史消息、不标记已读。
- 不处理图片、语音、附件、职位卡片等非文本消息的语义回复。

## 数据或状态变化

- 新增默认 `aiReply.prompt`，保存到现有 `local:web-geek-job-FormData` 配置。
- 聊天页运行时维护自动回复状态：是否处理中、最后生成草稿的消息 key、错误提示。
- `sessionStorage` 保存当前标签页已处理消息 key，key 由会话标题、消息时间、消息内容和页面 URL 组合。
- 自动回复请求模型前会只读获取当前 BOSS 简历预览摘要，并裁剪到固定长度作为 `chat.profile`。
- 自动回复请求模型前会从当前聊天页顶部岗位区域提取岗位名称、薪资、城市、公司/招聘者标题等可见文本，作为 `chat.job`。
- 如果当前聊天页存在 `查看职位` / `/job_detail/` 链接，自动回复会只读抓取该同源详情页 HTML，提取职位描述、技能标签、任职要求等正文摘要，并合并进 `chat.jobText`。
- 自动回复请求模型前会读取当前保存的 `aiFiltering.prompt` 和 `aiFiltering.score`，裁剪后作为 `chat.filteringCriteria`，用于让回复更贴合用户筛岗位标准。
- 不新增后端 API，不新增持久化数据库。

## API 或页面影响

- `/web/geek/chat` 模块初始化 `conf` 和 `model` store。
- 聊天记录悬浮面板显示自动回复状态，并提供“立即回复”按钮用于手动生成当前最新未处理消息的回复草稿。
- AI 配置页解锁 `AI回复` 开关和 Prompt 配置入口。
- 生成结果通过 DOM 填入当前 BOSS 聊天输入框，不调用 BOSS 发送接口。

## 验收标准

- 用户在 AI 配置中可以启用 `AI回复` 并保存 Prompt。
- 打开 `https://www.zhipin.com/web/geek/chat` 并选择会话后，聊天面板能显示自动回复状态。
- 对方新增一条文本消息后，插件在配置的延迟后调用已选模型生成回复草稿。
- 模型上下文包含当前会话最近 20 条消息、当前岗位卡片摘要、当前岗位详情页正文摘要、求职者资料摘要和 AI筛选标准。
- AI筛选标准进入上下文后，草稿仍必须输出普通聊天正文，不能输出 AI筛选 JSON。
- 对方问技术、版本、模块、项目、经验时，草稿必须先回答已有上下文中的具体信息；上下文没有答案时，要明确说“这块简历里没有展开/我需要确认”，不能只说“进一步沟通确认”。
- 对方要求发附件简历、联系方式或资料时，草稿不能只写“好的/可以的/没问题”，必须顺带获取一个岗位关键问题。
- 生成内容非空时，插件只把文本填入当前聊天输入框，不自动发送。
- 本人消息、系统消息、已生成过草稿的对方消息不会再次触发自动回复。
- `立即回复` 在没有识别到 `direction=other` 时，仍能对最新非本人、非系统的可见文本消息生成草稿；该兜底不用于自动挂机回复。
- 模型未配置、VIP 模式、发送渠道不可用或模型报错时，面板显示错误状态，不中断聊天页。
- `pnpm check`、`pnpm lint`、`pnpm build:chrome` 通过。
