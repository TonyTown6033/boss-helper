# Boss 聊天会话本地记录设计

## 目标

在 Boss 聊天页提供只读会话记录能力。用户打开某个会话后，插件记录当前页面已加载的消息内容，允许导出 JSON 和 CSV，方便用户在本地继续分析并手动操作。

## 当前缺口

- 现有页面模块只覆盖职位列表页，没有 `/web/geek/chat` 入口。
- 项目已有聊天消息发送和 Protobuf 相关基础，但没有聊天页历史消息采集和导出 UI。
- 当前无法把用户手动打开的聊天记录保存到本地文件。

## 已锁定决策

- 只记录用户当前打开的会话，不自动遍历联系人列表。
- 初版只读取 DOM 中已经加载出来的消息，不主动请求历史接口，不自动滚动加载更早消息。
- 用户可以手动滚动聊天区加载更多历史消息，再点击刷新采集。
- 采集过程只读，不发送消息、不标记已读、不点击联系人、不调用加好友或投递接口。
- 导出格式支持 JSON 和 CSV，文件名使用 `boss-chat-YYYYMMDD-HHmmss`。
- 采集结果只保存在当前页面运行时内存中，不写入浏览器 storage。
- 聊天页 UI 以固定悬浮面板展示采集数量、当前会话标题、刷新和导出按钮。

## 非目标

- 不绕过登录、验证码、风控或平台限制。
- 不做全量聊天记录爬取或联系人批量遍历。
- 不解析 WebSocket/Protobuf 实时消息。
- 不在初版调用 Boss 历史消息接口。
- 不做 AI 自动回复或自动操作。

## 数据或状态变化

- 运行时保存当前会话标题、页面 URL、采集时间和消息数组。
- 每条消息保存：序号、发送方、方向、文本内容、时间、头像地址、DOM 来源类名。
- 状态不持久化，刷新页面后重新采集。

## API 或页面影响

- `/web/geek/chat` 路由加载新的聊天采集模块。
- 页面右下角新增 `Boss 聊天记录` 悬浮面板。
- 面板提供“刷新采集”“导出 JSON”“导出 CSV”按钮。
- 无后端 API 变化。

## 验收标准

- 打开 `https://www.zhipin.com/web/geek/chat` 后能看到聊天记录采集面板。
- 用户选择一个会话并点击“刷新采集”后，面板显示已采集消息数。
- 已加载消息可以导出为 JSON 文件。
- 已加载消息可以导出为 CSV 文件，中文、逗号、双引号和换行不破坏列结构。
- 未选择会话或页面没有消息时，刷新采集不会报错，并显示 0 条。
- 功能不会发送消息、切换会话、自动滚动或主动请求历史接口。
