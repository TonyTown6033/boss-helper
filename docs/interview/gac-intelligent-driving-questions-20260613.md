# 广汽智驾感知岗位反问清单 - 2026-06-13

## 使用原则

不要一次性把所有问题抛出去。先问 `岗位侧重点`，再根据对方身份选择后续问题。

优先级：

1. 先确认岗位到底偏 `视觉算法研发`，还是偏 `车载部署调优 / 评测验证 / 数据闭环`。
2. 再确认技术栈和面试考察重点。
3. 最后确认流程、合同主体、薪资结构。

## 开场确认

如果刚加微信：

```text
何老师您好，我是 BOSS 上沟通过的梁效源。感谢您添加，我想先确认下这次沟通是广汽研究院的智驾感知算法开发岗位对吗？
```

如果对方让你先介绍自己：

```text
可以的。我先简单介绍一下，也想同步确认这个岗位的真实侧重点，方便我判断匹配度。
```

## 必问问题

### 1. 岗位侧重点

```text
这个岗位更偏视觉模型算法开发，还是偏车载平台部署调优、评测验证和数据闭环？
```

为什么问：

- 这是最关键问题。
- 如果偏模型训练，你匹配度较弱。
- 如果偏工程化落地，你可以主推 `Linux / DPDK / ONNX Runtime / 问题定位 / 交付闭环`。

### 2. 行车还是泊车

```text
这个岗位主要负责行车感知还是泊车感知？感知对象主要是车辆、行人、两轮车、车道线、障碍物，还是 BEV/融合感知？
```

为什么问：

- 判断后续技术准备方向。
- 行车、泊车、BEV、融合感知考察点不同。

### 3. 技术栈

```text
团队主要技术栈是 Python/PyTorch/ONNX/TensorRT，还是 C++ 车端部署更多？
```

为什么问：

- 如果提到 `ONNX/TensorRT/部署/评测`，你要重点讲 AI 工程化。
- 如果强调 `C++/车端框架/实时系统`，你要讲华为系统开发和 DPDK。

### 4. 面试重点

```text
后续面试会更看算法基础、项目深挖，还是工程落地和问题定位？
```

为什么问：

- 让对方提前暴露考察方式。
- 你可以据此准备 `DPDK CPU 告警定位`、`kpatch`、`ONNX Runtime`、`Keras 人脸识别`。

## 技术负责人可问

### 模型部署

```text
目前模型部署链路是 PyTorch 转 ONNX/TensorRT，还是有内部自研部署工具链？
```

```text
车端平台主要关注哪些指标？比如单帧延迟、吞吐、CPU/GPU/NPU 占用、功耗和温度？
```

### 评测验证

```text
感知评测是以离线数据集为主，还是会结合真值系统路测结果做闭环？
```

```text
bad case 闭环一般怎么做？是算法同学自己分析，还是有专门的数据和评测平台支持？
```

### 数据与标注

```text
数据采集和标注规范是团队自己制定，还是有集团统一平台和标准？
```

```text
标注验收主要看哪些维度？比如漏标、错标、框偏差、场景覆盖和一致性？
```

### 标定与图像质量

```text
摄像头下线标定这部分是岗位核心职责之一吗？团队更希望候选人做算法实现，还是做工具链和质量验证？
```

```text
图像质量问题，比如模糊、过曝、欠曝、雨雾、脏污和眩光，通常是算法团队负责分析，还是和硬件/测试团队一起闭环？
```

## HR / 招聘负责人可问

### 流程


```text
技术面大概几轮？会有笔试或手撕代码吗？
```

### 团队

```text
这个岗位所在团队是广汽研究院内部团队，还是和项目组/供应商有联合开发？
```

```text
岗位入职后主要对接算法、平台、测试，还是量产项目团队？
```

### 合同和薪资

```text
这个岗位是广汽研究院直招吗？劳动合同和社保主体是哪家公司？
```

```text
薪资结构是月薪 + 年终，还是包含绩效、项目奖或其他奖金？试用期薪资是否打折？
```

### 地点

```text
工作地点是在广州番禺广汽研究院吗？是否需要长期出差或到测试场/项目现场支持？
```

## 如果对方问你有什么问题

短版：

```text
我主要想确认三个点：第一，这个岗位更偏视觉算法模型开发，还是偏部署调优、评测验证和数据闭环；第二，主要技术栈是 Python/ONNX/TensorRT 还是 C++ 车端更多；第三，后续面试更看算法基础还是项目和问题定位。
```

更自然版：

```text
我想先确认下岗位侧重点。因为我之前更偏系统工程、Linux、AI 工程化和问题定位，不是完整智驾视觉量产背景。如果团队这边更需要模型部署调优、评测验证、数据闭环和工具链建设，我觉得我的经验会更匹配。所以想了解下实际工作里这几块占比大概怎样。
```

## 不建议问

不要一上来问：

```text
最低能给多少？
```

```text
能不能保证 40K？
```

```text
加班严重吗？
```

```text
我没有经验你们能不能培养？
```

这些问题不是不能问，而是时机要靠后。前期先让对方确认你有工程化价值。

## 红旗信号

如果对方出现这些情况，要降低预期：

- 说不清岗位是算法研发、部署调优还是评测验证。
- 说不清劳动合同和社保主体。
- 明确要求驻场供应商，但岗位包装成广汽研究院。
- 只强调“先面试再说”，但不愿透露技术栈和团队方向。
- JD 写广汽研究院，但实际沟通主体是第三方外包或人力公司。
